## 의미
내재적인(Implicit) + 피드백(Feedback)
=> 사용자가 직접적으로 드러내지 않고 간접적으로 드러내는 반응.
*예시: 구매 기록, 시청 기록*

### 핵심 특성
1. Negative Feedback이 없음
	: 사용자의 행동으로는 무엇을 싫어하는지 알 수 없다.
	ex) 어떤 프로그램을 사용자가 보지 않았다. 왜?
	- 싫어서?
	- 몰랐을지도?
	- 시간이 없었나?
	: 따라서, missing data까지 반드시 고려해야 한다.
2. Noisy한 데이터
	: 행동 데이터는 진짜 선호를 정확하게 반영하지 않는다.
	ex) 어떤 사용자가 물건을 구매했다.
	- 선물이었을까?
	- 본인이 쓰려는 걸까?
	- 잘못 샀을지도?
	: 관측된 행동과 선호는 일치하지 않는다.
3. 값의 의미가 다름
	: Implicit Feedback은 직접적인 선호보다는 confidence에 집중한다.
	ex) 어떤 사람이 이 영상을 많이 봤다.
	- 많이 봤다고 더 좋아한다는 보장은 없음.
	- 다만, 더 확신할 수는 있음.
4. 평가 방식이 다름
	: Implicit Feedback에서는 MSE 등의 기존 평가 방식이 적절하지 않다.
	- negative가 없음
	- 반복 소비 존재
	- 상품 사이의 경쟁 존재
