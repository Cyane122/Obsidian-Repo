# 텍스트 전처리
## 왜 전처리가 필요할까?
- 원본 텍스트를 분석하기 용이하도록 다듬을 필요가 있다.
	`ㅋㅋㅋ 진짜 맛있어요!!! 근데 배송이 너무 느리내요ㅜㅜ 3일은 좀...`
	- 특수문자가 섞임(!, $, @)
	- 같은 말, 다른 형태(먹다, 먹어, 먹었어요)
	- 쓸모없는 단어(은, 는, 이, 가 등)
	- 오탈자
	- 대소문자 혼용(Python vs python)
	`["맛있다", "배송", "느리다"]`
	

> [!전처리의 목표]
> 	컴퓨터가 오해하지 않도록 텍스트를 깨끗하고 일관적인 형태로 만드는 것.

	
## 전처리의 흐름
1. 정제: 노이즈 제거(특수문자, HTML 태그 등)
2. 정규화: 표현 통일(대소문자, 줄임말 등)
3. 토큰화: 단어/형태소 단위로 쪼개기
4. 불용어 제거(분석에 필요없는 단어 제거)

## 토큰화 (Tokenization)
- 토큰(Token): 분석의 최소 단위
- 토큰화(Tokenization): 텍스트를 토큰 단위로 쪼개는 작업.
	- 예시: `"나는 학교에 간다"`
	- 단어 토큰화: `["나는", "학교에", "간다"]`
	- 형태소 토큰화: `["나", "는", "학교", "에", "가", "ㄴ다"`
		-> 어떤 단위로 쪼개느냐에 따라 분석 결과가 달라진다.

### 영어 토큰화
- 영어는 띄어쓰기 단위로 단어가 쪼개지므로 토큰화가 간단하다.
- `split()`을 통해 문장을 단어 단위로 쪼갠다.
	- 구분자: 문자열을 자르는 기준.
	- 결과값: 쪼개진 문자열들이 담긴 리스트 형태.
	1. 기본 공백 분할: 별도의 인자를 주지 않았을 때.
		```Python
		text = "안녕하세요 반갑습니다 여러분"
		result = text.split() # ['안녕하세요', '반갑습니다', '여러분']
		```
	2. 특정 문자로 분할
		```Python
		text = "사과, 포도, 복숭아"
		result = text.split(', ') # ['사과', '포도', '복숭아']
		```
	- `split()`의 한계: 동일한 형태로 쪼개지는 경우가 아니라면 구분이 불가능함. => NLTK 라이브러리가 필요하다!

## 정규화
- `re.sub('정규식')`을 활용해 불필요한 텍스트를 정제할 수 있다.
- `konlpy`의 토크나이저들을 활용하여 명사, 동사, 형용사 등을 추출할 수 있다.

## NLTK
- NLTK 라이브러리의 `word_tokenize()` 함수를 이용하면 축약형, 구두점 등을 깔끔하게 정리해준다.
	`I'm a student.` -> `["I", "'m", "a", "student", "."]`
- `sent_tokenize()` 함수를 이용하면 문장을 깔끔하게 정리해준다.
