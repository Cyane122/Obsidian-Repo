## 🚀 STEP 0. 먼저 전체 코드를 실행해봅시다

이론 설명 전에 전체 파이프라인을 한 번 돌려봅니다. 지금은 이해하지 못해도 괜찮습니다. **결과가 나오는 것**을 먼저 경험합니다.

```python
# ─── 준비 ────────────────────────────────────────────────────
!pip install scikit-learn --quiet

import re
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.model_selection import train_test_split
from sklearn.naive_bayes import MultinomialNB
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score, classification_report
```

```python
# ─── 데이터 ──────────────────────────────────────────────────
긍정_리뷰 = [
    "치킨이 정말 바삭하고 맛있어요 소스도 최고예요",
    "배달이 엄청 빠르고 포장도 꼼꼼했어요",
    "피자 도우가 쫄깃하고 치즈가 넘쳐요 완전 맛집",
    # ... (25개)
]
부정_리뷰 = [
    "배달이 너무 늦어서 음식이 다 식었어요",
    "치킨이 눅눅하고 기름기가 너무 많아요",
    "포장이 엉망이라 국물이 다 쏟아졌어요",
    # ... (25개)
]

X = 긍정_리뷰 + 부정_리뷰   # 리뷰 텍스트
y = [1]*25 + [0]*25          # 정답 (1=긍정, 0=부정)
```

```python
# ─── 핵심 파이프라인 5단계 ────────────────────────────────────

# ① 전처리
def 전처리(텍스트):
    텍스트 = re.sub(r'[^가-힣a-zA-Z0-9 ]', ' ', 텍스트)
    return re.sub(r'\\s+', ' ', 텍스트).strip()

X = [전처리(r) for r in X]

# ② 데이터 분리
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42)

# ③ TF-IDF 변환
변환기 = TfidfVectorizer(analyzer='char_wb', ngram_range=(2, 4))
X_train_vec = 변환기.fit_transform(X_train)   # 훈련만 fit!
X_test_vec  = 변환기.transform(X_test)         # 테스트는 transform만!

# ④ 학습 + 예측
모델 = MultinomialNB()
모델.fit(X_train_vec, y_train)
예측 = 모델.predict(X_test_vec)

# ⑤ 평가
print(f"정확도: {accuracy_score(y_test, 예측):.2%}")
print(classification_report(y_test, 예측, target_names=["부정", "긍정"]))
```

## 🔍 STEP 2. 각 단계 이론 — ① 전처리

### 2-1. 코드 다시 보기

```python
def 전처리(텍스트):
    텍스트 = re.sub(r'[^가-힣a-zA-Z0-9 ]', ' ', 텍스트)
    return re.sub(r'\\s+', ' ', 텍스트).strip()
```

### 2-2. 왜 이렇게 하나?

```
입력: "치킨이 진짜 바삭하고 맛있어요!!! ㅋㅋ"
출력: "치킨이 진짜 바삭하고 맛있어요"
```

`[^가-힣a-zA-Z0-9 ]` 패턴의 의미:

```
^ = 이것이 아닌 것
가-힣 = 한글 전체
a-zA-Z = 영어 대소문자
0-9 = 숫자
(공백) = 공백

→ "한글, 영어, 숫자, 공백이 아닌 모든 것"을 공백으로 교체
```

### 2-3. 왜 KoNLPy(형태소 분석)를 안 쓰나?

3주차에서는 KoNLPy로 명사를 추출했습니다. 4주차에서는 쓰지 않습니다.

```
KoNLPy 방식:
  "바삭하고" → okt.nouns() → ["바삭"]
  장점: 정확한 명사 추출
  단점: 실행 환경에 따라 결과가 달라질 수 있고, 속도 느림

이번 방식 (char n-gram):
  "바삭하고" → 그대로 유지
  TfidfVectorizer가 알아서 "바삭", "삭하", "하고"로 쪼갬
  장점: 빠르고 안정적, KoNLPy 없어도 됨
```

> 💡 **현실적인 팁**
> 
> 실전에서 KoNLPy와 char n-gram 중 어떤 게 나을지 모르면 둘 다 해보고 성능이 높은 걸 씁니다. 이것을 **Ablation Study**라고 합니다. 정답은 데이터마다 다릅니다.

## 🔍 STEP 3. 각 단계 이론 — ② 데이터 분리

### 3-1. 코드 다시 보기

```python
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42)
```
### 3-2. 왜 나눠야 할까?

**시험 비유:**

```
[나쁜 방법]
  시험 문제를 미리 알고 공부 → 시험 100점
  → 진짜 실력이 아님

[올바른 방법]
  시험 문제 모르고 공부 → 시험에서 진짜 실력 측정

머신러닝도 동일:
  훈련 데이터: 모델이 공부하는 데이터
  테스트 데이터: 시험 문제 (학습에 절대 사용 금지!)
```

### 3-3. random_state=42 의미

```
random_state 없이:
  실행할 때마다 훈련/테스트 구성이 달라짐
  → 성능 결과가 매번 달라져서 비교 불가

random_state=42 고정:
  항상 같은 방식으로 분리
  → 팀원과 같은 결과 재현 가능
  → 42는 IT 커뮤니티의 농담(인생의 답=42)에서 비롯된 관례
```

### 3-4. 데이터 누수 (Data Leakage) — 가장 흔한 실수

```
[잘못된 순서 ❌]
  ① 전체 50개로 vectorizer.fit()  ← 테스트 정보가 IDF에 섞임!
  ② 그 다음 훈련/테스트 분리
  → 성능이 실제보다 좋게 측정됨 (속는 것!)

[올바른 순서 ✅]
  ① 먼저 훈련/테스트 분리
  ② 훈련(37개)으로만 vectorizer.fit()
  ③ 테스트는 transform()만 사용
```

> 💡 **현실적인 팁**
> 
> 데이터 누수는 실무에서도 매우 자주 발생하는 실수입니다. 성능이 갑자기 너무 좋게 나오면 누수를 의심하세요. "이 정보가 예측 시점에 실제로 존재하는가?"를 항상 확인하는 습관이 중요합니다.

## 🔍 STEP 4. 각 단계 이론 — ③ TF-IDF 변환

### 4-1. 코드 다시 보기

```python
변환기 = TfidfVectorizer(analyzer='char_wb', ngram_range=(2, 4))
X_train_vec = 변환기.fit_transform(X_train)   # 훈련만 fit!
X_test_vec  = 변환기.transform(X_test)         # 테스트는 transform만!
```

### 4-2. analyzer='char_wb' + ngram_range=(2,4)

**n-gram이란?** 연속된 n개의 문자(또는 단어) 조합입니다.

```
"바삭하고" → char n-gram 분해:

2글자: "바삭", "삭하", "하고"
3글자: "바삭하", "삭하고"
4글자: "바삭하고"
```

**한국어에서 char_wb가 효과적인 이유:**

```
한국어는 같은 의미도 형태가 다양합니다:
  맛있다 / 맛있어요 / 맛있었어요 / 맛있고

공통 패턴 "맛있"이 항상 등장합니다!
→ KoNLPy 없이도 형태 변화를 자연스럽게 처리
→ 수천 개의 n-gram 패턴이 특성으로 추출됨
```

### 4-3. fit_transform vs transform

```
fit_transform(훈련 데이터):
  fit     = 어휘 목록과 IDF를 학습
  transform = 학습한 기준으로 벡터로 변환
  → 두 작업을 한 번에

transform(테스트 데이터):
  이미 학습된 어휘와 IDF를 그대로 사용해서 변환만
  → fit 없이 변환만
```

> 💡 **현실적인 팁**
> 
> `fit_transform`과 `transform`의 구분은 sklearn의 모든 전처리 도구에서 동일하게 적용됩니다. `StandardScaler`, `LabelEncoder` 등도 똑같이 훈련 데이터에만 `fit()`을 씁니다. 이 패턴을 한 번 익히면 어떤 도구에서도 동일하게 적용됩니다.

### 5-1. 나이브 베이즈 (MultinomialNB)

```python
모델 = MultinomialNB()
모델.fit(X_train_vec, y_train)
예측 = 모델.predict(X_test_vec)
```

**핵심 아이디어:** 각 단어(패턴)가 긍정/부정 리뷰에 등장하는 확률을 학습합니다.

```
학습 결과 (예시):
  패턴       긍정에서 확률   부정에서 확률
  "맛있"       0.80           0.10
  "바삭"       0.75           0.08
  "식었"       0.05           0.70
  "별로"       0.03           0.65

새 리뷰 "바삭 맛있다" 예측:
  P(긍정) ∝ 0.75 × 0.80 = 0.60  → 높음!
  P(부정) ∝ 0.08 × 0.10 = 0.008 → 낮음
  → 긍정으로 분류
```

**"나이브(순진한)"의 의미:**

각 단어가 서로 독립적이라고 가정합니다. 현실과 맞지 않지만 계산이 단순하고, 놀랍게도 잘 작동합니다.

**핵심 아이디어:** 각 패턴에 가중치를 부여해서 점수를 계산합니다.

점수 → 시그모이드 함수 → 0~1 확률
확률 ≥ 0.5 → 긍정 / 확률 < 0.5 → 부정

## 🔍 STEP 6. 각 단계 이론 — ⑤ 성능 평가 깊게 보기

### 6-1. 정확도만으로는 부족한 경우

```
예시: 리뷰 100개 중 긍정 95개, 부정 5개

모델이 모든 것을 "긍정"이라고 예측하면?
  → 정확도 = 95/100 = 95%  ← 높아 보이지만
  → 부정 리뷰를 단 하나도 못 잡아냄!

→ 정확도 하나로는 속을 수 있음
→ Precision, Recall, F1 을 함께 봐야 함
```

### 6-2. 혼동 행렬 (Confusion Matrix)

```
                  예측: 부정(0)   예측: 긍정(1)
실제: 부정(0)  [   TN (맞음)  |   FP (오탐)  ]
실제: 긍정(1)  [   FN (미탐)  |   TP (맞음)  ]

TN: 부정을 부정으로 ✅
TP: 긍정을 긍정으로 ✅
FP: 부정인데 긍정으로 ❌ → 불만 고객을 만족으로 착각
FN: 긍정인데 부정으로 ❌ → 만족 고객을 불만으로 착각
```

```python
from sklearn.metrics import confusion_matrix

cm = confusion_matrix(y_test, 예측)
print("혼동 행렬:")
print(f"  실제 부정 → 예측 부정(✅): {cm[0][0]},  예측 긍정(❌): {cm[0][1]}")
print(f"  실제 긍정 → 예측 부정(❌): {cm[1][0]},  예측 긍정(✅): {cm[1][1]}")
```

### 6-3. Precision / Recall / F1

```
           TP
Precision = ────────     "긍정 예측 중 진짜 긍정 비율"
           TP + FP

          TP
Recall = ────────         "실제 긍정 중 놓치지 않은 비율"
         TP + FN

         2 × P × R
F1 = ──────────────      "Precision과 Recall의 균형"
         P + R
```

💡 **현실적인 팁 — 언제 무엇이 중요한가?**

| 상황       | 중요 지표         | 이유                  |
| -------- | ------------- | ------------------- |
| 스팸 메일 필터 | **Precision** | 정상 메일을 스팸함에 넣으면 안 됨 |
| 암 진단     | **Recall**    | 암 환자를 놓치면 치명적       |
| 리뷰 분류    | **F1**        | 균형 있게 보는 것이 적합      |

### 6-4. classification_report 완전 해석

```
              precision    recall  f1-score   support
        부정       0.86      0.86      0.86         7
        긍정       0.83      0.83      0.83         6
    accuracy                           0.85        13
   macro avg       0.85      0.85      0.85        13
weighted avg       0.85      0.85      0.85        13
```

```
support = 7 → 테스트에 실제 부정 리뷰 7개 있음

부정 precision 0.86 → "부정"으로 예측한 것 중 86%가 실제 부정
부정 recall 0.86    → 실제 부정 7개 중 86%(≈6개)를 찾아냄

macro avg    → 두 클래스의 단순 평균
weighted avg → 두 클래스의 support 비율 가중 평균 (클래스 불균형 시 더 신뢰)
```

### 7-1. 어떤 패턴이 중요했나?

feature_names = 변환기.get_feature_names_out()
가중치 = 모델_LR.coef_[0]

긍정_순서 = 가중치.argsort()[-10:][::-1]

print("긍정에 강한 패턴:")
for i in 긍정_순서:
    막대 = "█" * min(int(가중치[i] * 3), 15)

> 💡 **현실적인 팁**
> 
> 가중치가 이상하게 나왔다면 (예: "맛없다"가 긍정 패턴에 포함) 데이터에 문제가 있을 가능성이 높습니다. 가중치 확인은 모델 디버깅의 첫 번째 단계입니다.

## 🔍 STEP 8. 새 리뷰 예측 + 모델의 한계

### 8-1. 예측 함수

```python
def 예측하기(새_리뷰, 모델):
    처리  = 전처리(새_리뷰)                   # ① 전처리
    벡터  = 변환기.transform([처리])           # ② 변환 (transform만!)
    예측값 = 모델.predict(벡터)[0]             # ③ 예측
    확률값 = 모델.predict_proba(벡터)[0]       # ④ 확률
    결과  = "긍정 😊" if 예측값 == 1 else "부정 😞"
    return 결과, 확률값[1]                     # 긍정 확률 반환

# 사용 예시
결과, 확률 = 예측하기("치킨이 바삭하고 완전 맛있어요", 모델)
print(f"{결과}  (긍정 확률: {확률:.1%})")
```

### 8-2. 확률 해석

```
긍정 확률 0.93  → 강하게 긍정이라고 확신
긍정 확률 0.55  → 애매함 (경계선 근처)
긍정 확률 0.07  → 강하게 부정이라고 확신
```

### 8-3. 모델이 잘 못하는 경우

```
✅ 잘 분류:
  "치킨이 바삭하고 완전 맛있어요"  → 긍정 93%
  "배달 3시간, 음식 다 식었어요"  → 부정 96%

🤔 헷갈림:
  "맛은 괜찮은데 배달이 느려요"         → 혼합 감성
  "가격은 비싸지만 그만한 가치는 있어요" → 부정처럼 보이지만 긍정
  "뭐... 그냥 그래요"                    → 중립
```

**근본 원인:** TF-IDF는 단어의 빈도만 봅니다. "맛있다"와 "맛없다"를 거의 같은 패턴으로 처리합니다.

> 💡 **현실적인 팁**
> 
> 실제 서비스에서는 모델이 확신하지 못하는 경우(확률 0.4~0.6 사이)를 별도로 분류해서 사람이 검토하게 합니다. 이를 **Human-in-the-loop** 방식이라고 합니다.

## 📌 전체 요약

### 오늘 배운 파이프라인

```
리뷰 텍스트
  ↓ ① 전처리()                특수문자 제거
  ↓ ② train_test_split()      훈련 75% / 테스트 25%
  ↓ ③ TfidfVectorizer         char_wb + ngram(2,4) → 수천 개 특성
     .fit_transform(훈련)      훈련만 fit!
     .transform(테스트)        테스트는 변환만!
  ↓ ④ 모델.fit()              나이브 베이즈 or 로지스틱 회귀
     모델.predict()
  ↓ ⑤ classification_report() 정확도 + Precision + Recall + F1
```

### 핵심 개념 한 줄 요약
|개념|한 줄 설명|현실 팁|
|---|---|---|
|레이블|모델에게 알려주는 정답 (1/0)|레이블 품질이 성능의 절반|
|훈련/테스트 분리|보지 않은 데이터로 진짜 성능 측정|80:20 또는 75:25가 일반적|
|데이터 누수|테스트 정보가 학습에 섞이는 반칙|성능이 너무 좋으면 의심하라|
|char n-gram|2~4글자 조합으로 특성 추출|KoNLPy 없이도 한국어 잘 됨|
|fit_transform|훈련 데이터: 학습 + 변환|테스트엔 절대 fit 금지|
|나이브 베이즈|단어 확률로 분류, 빠름|데이터 적을 때 먼저 시도|
|로지스틱 회귀|단어 가중치로 분류, 해석 가능|가중치로 모델 디버깅 가능|
|Precision|예측의 정확성|오탐이 치명적일 때 중요|
|Recall|실제를 빠짐없이 찾기|미탐이 치명적일 때 중요|
|F1|두 지표의 균형|일반적 상황에서 기본 지표|

### 핵심 코드

```python
# ① 전처리
def 전처리(텍스트):
    텍스트 = re.sub(r'[^가-힣a-zA-Z0-9 ]', ' ', 텍스트)
    return re.sub(r'\\s+', ' ', 텍스트).strip()

# ② 분리
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.25, random_state=42)

# ③ TF-IDF
변환기 = TfidfVectorizer(analyzer='char_wb', ngram_range=(2, 4))
X_train_vec = 변환기.fit_transform(X_train)
X_test_vec  = 변환기.transform(X_test)

# ④ 학습 + 예측
모델 = MultinomialNB()
모델.fit(X_train_vec, y_train)
예측 = 모델.predict(X_test_vec)

# ⑤ 평가
print(classification_report(y_test, 예측, target_names=["부정","긍정"]))
```
